memory and set a pointer named "s" to point to them. Then it reserves
another block of memory to store "This is a test!" and points "s" to that.
This means the block of 128 bytes of memory that were originally allocated is
now sitting empty and unusable, and the program is actually accessing the
memory that stores "This is a test!".
This will seem to work for a while, until the program tries to store more
bytes into that block than can fit into the 16 bytes reserved for "This is a
test!". Since C is poor about bounds checking, this may cause all kinds of
trouble.
