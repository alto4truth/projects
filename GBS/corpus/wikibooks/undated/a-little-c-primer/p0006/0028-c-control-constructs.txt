This is somewhat clumsy. The "switch" statement does a cleaner job:

The "switch" statement tests the value of a single variable; unlike the "if"
statement, it can't test multiple variables. The optional "default" clause
is used to handle conditions not covered by the other cases.
Each clause ends in a "break", which causes execution to break out of the
"switch". Leaving out a "break" can be another subtle error in a C program,
since if it isn't there, execution flows right through to the next clause.
