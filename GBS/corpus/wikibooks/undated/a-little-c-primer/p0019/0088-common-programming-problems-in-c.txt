The following will not do the job:

Remember that a function may require a pointer as a parameter even if it

14: Getting mixed up when using nested "if" and "else" statements. The best
way to avoid problems with this is to always use brackets. Avoiding
complicated "if" constructs is also a good idea; use "switch" if there's any
choice in the matter. Using "switch" is also useful even for simple "if"
statements, since it makes it easier to expand the construct if that is
necessary.
