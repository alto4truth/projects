==== notes ====
It may seem desirable to store heterogeneous data in parallel
arrays in what amounts to a FORTRAN-like style. This style is
an artifact of FORTRAN's data structuring limitations. FORTRAN
only has facilities for constructing homogeneous arrays.

==== exceptions ====
If the application must interface directly to hardware, and the
hardware requires that information be distributed among various
locations, then it may not be possible to use records.

=== Heterogeneous Polymorphic Data ===

==== guideline ====
Use access types to class-wide types to implement heterogeneous polymorphic data structures.
Use tagged types and type extension rather than variant records (in combination with enumeration types and case statements).
