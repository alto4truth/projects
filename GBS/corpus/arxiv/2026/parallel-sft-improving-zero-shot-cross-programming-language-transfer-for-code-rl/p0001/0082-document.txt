Code validation.We use the CodeForces submissions dataset. We use C++ and Python as our source languages
because they have the largest amount of data in this dataset. For these two PLs, we manually perform a train-test split.
We use batch size 32 and notice that the performance plateaus early and only report performance for 200 steps. We
train with all distinct samples for these 6,400 instances. For evaluation, we use the remaining instances for each PL.
To prevent class imbalance from skewing the accuracy, we manually balance the correct and incorrect solutions via
truncation, on a per-PL basis. The statistics of the final evaluation instances are: Python: 19996 samples, Go: 1502
samples, PHP: 992 samples, Ruby: 1150 samples, Java: 75194 samples, C++: 19998 samples, Rust: 534 samples,
Javascript: 2440 samples, C#: 8074 samples.
For both settings, we use a learning rate of5×10 −7.
