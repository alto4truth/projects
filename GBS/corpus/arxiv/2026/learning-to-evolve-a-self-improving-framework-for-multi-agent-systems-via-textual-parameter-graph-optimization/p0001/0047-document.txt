Optimization memory and rollback.GRAO
maintains a persistent optimization experience
memory. Each entry records the problem context, the proposed graph edit, its rationale, and the
observed effectiveness score. When a new error
cluster is encountered, candidate experiences are
retrieved by semantic similarity and re-ranked by
effectiveness; the highest-ranked positive and negative exemplars are then used as in-context demonstrations for the Optimizer LLM. After each iteration, we monitor validation performance and roll
back to the previous graph if an update degrades
performance on the targeted validation subset. Unsuccessful updates are still retained in memory. For
reproducibility, we archive all optimization artifacts, including proposals, graph states, and run
metadata, for every iteration.
