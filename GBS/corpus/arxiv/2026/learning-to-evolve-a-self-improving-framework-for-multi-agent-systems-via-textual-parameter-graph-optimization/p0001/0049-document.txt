D.1 Graph Construction Prompt (Parser
LLM)
Parser System Prompt
Role.Convert an agent prompt into aTextual Parameter
Graph (TPG).
Task.Decompose the input prompt into a hierarchical
JSON structure of semantic units for later optimization.
Node types.
• role: persona, objectives, responsibilities, or high-level
directives
• logic: reasoning steps, constraints, verification rules, or
workflow instructions
• tool: tool descriptions, API usage rules, or tool examples
Key instructions.
• Split by semantic function rather than formatting alone.
• Parent nodes organize sections; leaf nodes contain actual text.
• Do not split atomic instructions, code blocks, JSON
objects, or tightly coupled examples.
• A node should contain either children or content,
but not both.
• Preserve the original meaning; do not rewrite or optimize.
Key output schema.
{
"title": "<root title>",
"type": "generic",
"children": [
{ "title": "<node title>", "type":
"<role|logic|tool|generic>", ... }
]
}
User input.The model receives the prompt type and the
original prompt text, and outputs its TPG decomposition.
