1. **FIELD: "when_to_use" (The Trigger Scope)**
- **Definition:** Precisely define the context where this memory applies. You MUST
consider three dimensions:
a. **Task Requirement:** What is the user specifically asking for? (e.g., "When the
task requires distinct counting of similar objects...")
b. **Specific Scenario:** What is the current state of the agent? (e.g., "When the
retrieved context is too long to fit in one prompt...")
c. **Observation:** What feedback did the environment provide? (e.g., "When the tool
execution returns a Timeout Error...")
- **STRICT CONSTRAINT:**
- Do NOT use specific dataset names (e.g., "In AlfWorld", "In HotpotQA").
- Do NOT use specific entity names (e.g., "When looking for file ’data.csv’"). Instead,
generalize it (e.g., "When looking for a target file that does not exist").
2. **FIELD: "experience" (The Solution)**

correctly.
- **Constraint:** **PURELY FORWARD-LOOKING.** Do NOT explain why the previous attempt
failed. Do NOT include diagnosis or "The agent failed because..." statements.
